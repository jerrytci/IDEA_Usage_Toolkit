# 📖 IDEA Usage Toolkit 使用指南

`IDEA Usage Toolkit` 为 VS Code 带来了 IntelliJ IDEA 经典的“显示用法” (Show Usages) 体验。通过极简的 QuickPick 界面，让你在无需切换侧边栏的情况下，快速预览并跳转到任何方法、类或符号的调用位置。

---

## 1. 快速上手

### 触发方式
*   **快捷键触发**：将光标停留在方法名或类名上，按下 `Alt + X` (Windows/Linux)。
*   **鼠标触发 (实验性)**：按住 `Ctrl` 并点击符号名称。
*   **右键菜单**：在编辑器中点击右键，选择 `Show Usages (IDEA Style)`。

---

## 2. 核心特性

### 2.1 扁平化列表 (Flattened List)
不同于 VS Code 传统的按文件分组模式，我们采用**全平铺模式**：
*   **源码直观展示**：每一行直接显示调用处的代码片段。
*   **即时语义过滤**：由于通过图标区分，你可以快速在一大堆搜索结果中一眼扫到你想要的逻辑。

### 2.2 实时同步预览 (Live Sync Preview)
这是该插件最强大的功能之一：
1.  弹出搜索框后，使用 **键盘方向键 (↑/↓)** 切换选项。
2.  底部的编辑器会**实时同步跳转**到该位置并高亮。
3.  **零污染跳转**：预览过程中不会在编辑器顶部堆积多余的标签页。

### 2.3 现场保护与回滚 (Context Safety)
*   **确认跳转**：按下 `Enter` 键确认并跳转。
*   **无感回滚**：按下 `Esc` 键退出或点击搜索框外部，编辑器会自动**瞬间滚回到你发起搜索前的原始位置**。这极大提升了在复杂代码库中“看一眼就回来”的效率，再也不怕切屏切迷路。

---

## 3. 常见问题记录

*   **Q: 为什么按快捷键没反应？**
    *   插件仅在“调用型符号”（如方法、类、接口）上生效。如果是普通变量，会自动退回到 VS Code 原生的引用视图。
*   **Q: 是否支持 Java/TS/Python？**
    *   只要你的 VS Code 安装了对应的语言扩展（如 Language Support for Java），本插件就能利用语言服务器的数据提供搜索。

---

## 4. 推荐配置
为了获得极致体验，建议在 `keybindings.json` 中将原生的引用搜索禁用，完全由本插件接管：
```json
{
    "command": "-editor.action.referenceSearch.trigger",
    "key": "alt+shift+f12",
    "when": "editorTextFocus"
}
```
*(注：本插件已默认覆盖了 Alt+X，无需手动配置。)*
